Actividad 4: Instalación y Configuración de Docker Engine

Instalación de Docker Engine nativo en Linux (Kubuntu), creación de una imagen personalizada basada en **Debian** mediante Dockerfile y gestión básica del ciclo de vida de un contenedor.

4.1 Instalación de Docker Engine en Kubuntu

Pasos de Instalación y Configuración

Se configuró el repositorio oficial de Docker [cite: 111], se instalaron los paquetes requeridos [cite: 112] y se agregó el usuario actual al grupo `docker` para evitar usar `sudo`[cite: 113]. El servicio fue habilitado con `systemctl enable docker`[cite: 114]. 

                # Comandos de instalación y configuración
                sudo apt install docker-ce docker-ce-cli containerd.io -y
                sudo usermod -aG docker ${USER}
                systemctl enable docker
                docker run hello-world # [cite: 115] Verificación

4.2 Creación de Imagen Docker (Dockerfile Debian)

Dockerfile Comentado

Se creó una imagen personalizada utilizando **Debian** como base, instalando herramientas de desarrollo (`git`, `curl`) y configurando un usuario no root[cite: 127, 128]. 

                # [cite: 131] Archivo Dockerfile
                FROM debian:stable-slim # Base Debian [cite: 126]
                RUN apt update && apt install -y git curl nano make gcc [cite: 127]
                
                RUN useradd -ms /bin/bash appuser # [cite: 128] Configurar usuario no root
                USER appuser
                
                WORKDIR /home/appuser/app
                CMD ["/bin/bash"]

4.3 Creación y Gestión de Contenedor

Ciclo de Vida y Comandos

Se demuestra el ciclo de vida del contenedor utilizando comandos clave: `docker run`, `docker ps`, `docker exec`, `docker logs`, `docker stop` y `docker rm`[cite: 138]. Se incluye una inspección con `docker inspect`[cite: 139]. 

                docker run -it --name dev-shell mi-imagen:v1.0
                docker ps -a
                docker exec -it dev-shell /bin/bash
                docker stop dev-shell
                docker rm dev-shell
